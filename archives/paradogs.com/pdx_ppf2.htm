Introduction
 
What is PPF?

PPF (PlayStation Patch File) is a tool dedicated to all PlayStation coders and developers out there who are creating PAL/NTSC patches, trainer options and even cracks for your favourite console system. With the files in the PPF package you are in the position to make patchfiles similar to IPS on SuperNES. Finally there is no need for writing own .EXE files for applying the patches to ISO-images anymore - just let PPF do this job for you.

PPF-O-Matic is a GUI (Graphics User Interface) for our very own PPF (PlayStation Patch File) system. It was created after having received thousands (!) of requests as it seems that typing in a simple line in MS-Dos is too hard for today's computer kid generation.
 
Why PPF?

Before PPF hit the scene all the groups producing PAL/NTSC fixes and even trainer versions for the PlayStation included selfmade .exe files for PCs and compatibles in order to apply the patch. There is one big problem: different operating systems (OS). Many sceners out there are using Linux, AmigaOS, MacOS, BeOS or whatever else they prefer to run their systems. Those people were not able to use those patches.... until now as with PPF the patchfiles are no executable files anymore.
Anyone with the for the his system compiled PPF package is able to apply the patches. You can find the C-sourcecode in the v1.0 archive to compile the PPF system for all possible platforms.
 
How to use?
 
MakePPF.exe
What is MakePPF.exe

With MakePPF.exe you are able to create own PPF2.0 (and only PPF2.0) patches. This is part of the PPF2.0 dos distribution! If you really dont know what MakePPF is used for and you still think that it is some kind of "autocracker" (yes! we get tons of e-mails about this autocracker-shit), stop reading now and don't touch MakePPF!

How to use MakePPF.exe

To create a PPF-File is simple.. You have to enter a very easy commandline like this e.g.:

MakePPF [Original_ISO.bin] [Patched_ISO.bin] [PPF_FILE-Output]

Original_ISO is supposed to be the original untouched CDRWin binfile. Patched_ISO is the the patched (touched ;)) CDRWin binfile. PPF_FILE will be the filename of the PPF-output.

Please make sure that the filesize of the original and patched ISO files are identical!

After entering the correct commandline MakePPF.exe asks you for a PPF-patch description. Enter a short explaination e.g. about the patch features or something useful to identify the patch. You have a maximum of 50 chars for the description.

Allright. There you go. Just wait and the PPF-file is ready to travel around the globe.

Final words

After having successfully created a PPF2.0 patch with MakePPF.exe please keep in mind to add a file_id.diz using PPFDiz.exe. Just to make it clear for everyone: this is not necessary but what's a scene release without file_id.diz?!? :)
 
ApplyPPF.exe
What is ApplyPPF.exe

ApplyPPF.exe is part of the PPF2.0 dos distribution! With ApplyPPF.exe you are able to apply PPF1.0 and PPF2.0 images over binfiles (e.g. PSX-CDRWin-Bins). At this point of time everyone should know now what PPF patches are, so there is really no need for a further introduction.

How to use ApplyPPF.exe

Applying a PPF-patch is a very simple job.

Following please find a simple commandline example:

ApplyPPF.exe MyBinfile.bin MyPPFPatch.ppf

Thats it! - If the PPF-patch is a v2.0 patch and includes a file_id.diz it will be shown then after entering the commandline! So you are able to see some additional patch information added by the creators of the patch.

PF2.0 is supporting a special image and filesize check. Now your binfile will be verified before patching to avoid applying patches on false binfiles.

It may happen that your binfile is not the right one. It is possible that two warning messages can or better will pop up. Here's a description in detail:

First warning:
"The filesize of the binfile does not match with the filesize of the binfile this PPF-Patch was made of! Continue? (y/n)"

What is wrong:
It seems you have not the correct binfile for successfull patching. If you continue it's most likely that the binfile will be corrupt afterwards and the game won't work after burning the CD. With only this warning on your screen you still have a chance to 50:50 that it might work afterwards. Various CDRWin versions may add some more bytes to the binfile so this warning has not to be taken too serious!

Second warning:
"The binfile does not to be the same one this PPF-Patch was made of! Continue? (Suggestion: no!) (y/n)"

At this point when the second warning screen appears you have got a serious problem. The binfile you'd like to patch was not identified to be the correct binfile for this special PPF-patch! It's most likely that you will burn a nice coaster if you continue burning! Make sure to organise yourself the correct version of the game you'd like to be patched. Sometimes rental, store or beta versions can cause problems.

That are all information you need to know about ApplyPPF.exe.

Final words

If you have still have problems using a DOS-command then better have a look at PPF-O-MATIC V2.0, a WIN95 GUI for both MakePPF.exe and ApplyPPF.exe. This one gives a helping hand to you.
 
PPFDiz.exe
What is PPFDiz.exe

PPFDiz.exe is a part of the DOS-distribution of PPF2.0. It is used to add a file_id.diz to PPF2.0 patches. Furthermore you have an option to view a file_id.diz within a PPF2.0.

How to use PPFDiz.exe

- Add a File_id.diz:

The commandline for adding a file_id.diz id very simple. Use pption "a". Here is a simple example:

PPFDiz.exe a [PPF2.0-File] [FILE_ID.DIZ]

- Show a File_id.diz:

If you want to show a file_id.diz within a PPF2.0-patch use the option "s". Here is a simple example:

PPFDiz.exe s [PPF2.0-File]

Final words

Please be sure to add a file_id.diz to every PPF2.0 patch you distibute via the Internet. It simplyfies the identification of the file. Furthermore you can use it for your own comments on the patch etc...
If you use PPF-O-MATIC v2.0 for creating the patches you don't need this one because there is an option to add this while doing the patch! PPFDiz.exe is for the DOS-fanatics out there!
 
PPFInfo.exe
What is PPFInfo.exe

PPFInfo.exe is a part of the DOS distribution of PPF2.0 It is used to show various information about a PPF2.0 Patch.

How to use PPFInfo.exe

The commandline for viewing the patch info is very simple. Here is a simple example:

PPFInfo.exe MyPPFpatch.ppf

This will will show the following information: 

- Filename      : (guess)
- Filelength    : (gues)
- Enc.Method    : (what encoding method is used for the patch)
- Description   : (the PPF-Patch description)
- File_id.diz   : (shows the PPF2.0 file_id.diz)
- File_id length: (length of the file_id.diz)
- PPF Patchdata : (length of the actual patchdata)

Detailed File Structure
PPF 2.0 header  
.----------+--------+---------------------------------------------.
| POSITION |  SIZE  |              E X P L A N A T I O N          |
+----------|--------|---------------------------------------------+
| 00-04    |   05   | PPF-Magic: "PPF20"                          |
+----------|--------|---------------------------------------------+
| 05       |   01   | Encoding Method:                            |
|          |        | - If $00 then it is a PPF 1.0 Patch         |
|          |        | - If $01 then it is a PPF 2.0 Patch         |
+----------|--------|---------------------------------------------+
| 06-55    |   50   | Patch Description                           |
+----------|--------|---------------------------------------------+
| 56-59    |   04   | Size of the file (e.g. CDRWin binfile) this |
|          |        | patch was made of. Used for Identification  |
+----------|--------|---------------------------------------------+
| 60-1083  | 1024   | this is a binary block of 1024 byte taken   |
|          |        | from position $9320 of the file (e.g. CDRWin|
|          |        | binfile) this patch was made of. Used for   |
|          |        | identification.                             |
+----------|--------|---------------------------------------------+
| 1084-X   |   XX   | The Patch itself.. see below for structure! |
'----------+--------+---------------------------------------------'

TOTAL HEADER-SIZE = 1084 BYTES
 
Patchfile Structure
PPF 2.0 patch itself (encoding method #1)  
FORMAT: xxxx,y,zzzz

         xxxx   = 4 byte file offset.
    
         y      = Number of bytes that will be changed.

         zzzz   = New data to be written ('y' number of bytes).
 
Examples
Starting from File Offset 0x0015F9D0 replace 3 bytes with 01,02,03
D0 F9 15 00 03 01 02 03

Remember to be careful. Have a look at the endian format!!! Amiga coders attempting to compile the PPF2-Patcher for Amiga should not forget to swap the endian-format of the offset to avoid seek errors!
 
Patchfile File-ID Area
The PPF 2.0 file-ID area

If you are interested in storing additional patch information the PPF 2.0 file ID area is the the place where you dreams come true. The Amiga file_id.diz standart has been implanted.
You are able to add a file_id to a PPF 2.0 patch by using the tool "PPFdiz.exe" or "PPF-O-MATIC2", both included in this package. You don't have to add a file_id to your PPF 2.0 patch. It's just for your pleasure! :)

For developers: a file_id area begins with @BEGIN_FILE_ID.DIZ and ends with @END_FILE_ID.DIZ (Amiga BBS standard). Between @BEGIN_FILE_ID.DIZ and @END_FILE_ID.DIZ you will find the File_Id and followed after @END_FILE_ID.DIZ you will find an integer (4 byte long) with the length of the FILE_ID.DIZ!

A File_ID.diz file can not exceed 3072 bytes.

If you do a PPF 2.0 Applier be sure to check for an existing file ID area, because it is located after the patch data!
 
Developers wanted

In order to develope the PPF standard further we invite all interested people to create utilities, a GUI (graphical user interface), a description lister or whatever else might be useful. Only the effort of an entire community can develope this one further.
Maybe someone can develope a better encryption method. Use your illusion. Any useful expansions, tools and GUIs will be offered here for download, ofcourse containing your well deserved credits. Become a part of the developers team and share your skills with us. Remember to send in all your developments regarding PPF. Many thanks in advance.

PPF has been designed with several features as standard.

- No file limitations
PPF facilitate patching of ISO-files up to 2 GB

 

- Internal Descriptors
A small text field allowing PPF authors to describe precisely what the patch does

 

- Future expansion through different patch types and encoding methods

Included in the archive you can find the compiled Amiga and PC versions of both important files, ApplyPPF.exe and MakePPF.exe.
Furthermore you can find C-sources included for our Linux/Unix users. If you want to adapt it for other operating systems feel free to do it.

If you are interested in learning more about the PPF file structure please read the document "PPF.txt" carefully. It is included in the archive you can download here.
