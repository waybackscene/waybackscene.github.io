Pour les besoins de cette section, un petit Crackme (avec source) nommé 'RiDiCuLe' a été créé afin de servir de référence de code non protégé. Il est donc facile de le tracer et de se familiariser avec celui-ci. Si le coeur vous en dit, vous pouvez même rechercher un mot de passe valide... ;)

Les anti-debugger présentés dans cette section sont en général facilement neutralisables. Le but n'étant pas de cacher mais de montrer.

L'Art est bien sûr de jongler avec les anti-debugger tout en les rendant les plus furtifs possible. Se battre contre les crackeurs demande-t-il une certaine dose de perversité ? possible...

Tous les anti-debugger sont appliqués à RiDiCuLe afin que vous puissiez voir concrètement l'action de ceux-ci.
