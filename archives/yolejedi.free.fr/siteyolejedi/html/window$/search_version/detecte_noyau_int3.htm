Detection du Noyau
-=¤ int 3 ¤=-
(version 1.01)
par YoLeJedi

 

Il est étrange de constater que window$ ne fournie pas les mêmes informations à un SEH losqu'une interruption 3 (EXCEPTION_BREAKPOINT) est déclenchée.
En effet, le noyau 9x interprète cette interruption en fournissant au SEH une interruption de type trap. Le registre EIP dans le CONTEXT pointe donc sur l'instruction qui suit immédiatement l'instruction qui a déclenché l'interruption 3 : int 3 (0xCC).
Par contre, le noyau NT respecte le type de l'interruption 3 et fournie donc au SEH une interruption de type faute. Le registre EIP dans le CONTEXT pointe donc sur l'instruction qui a déclenché l'interruption 3 : int 3 (0xCC).
